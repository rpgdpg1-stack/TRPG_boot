# TRPG UI

Дизайн-система проекта **TRPG**. Источник истины — `src/index.css` (CSS-переменные).
Стили — НЕ Tailwind: inline-объект `styles` внизу файла компонента + классы из `index.css`
для псевдосостояний (press/hover/анимаций). Иконки — inline SVG с `currentColor`.

## Железное правило

**Не хардкодить значения — брать из CSS-переменных.** Радиусы, цвета, шрифты, тайминги
заданы в `index.css`. Если в правке появляется `borderRadius: '20px'` или `color: '#888'` —
заменить на `var(--radius-medium)` / `var(--color-text-secondary)`.
(В проекте уже есть рассинхрон — хардкод `33px`, `#222222` вместо переменных. Новый код пишем чисто,
старый чиним по ходу, когда трогаем файл.)

## Скругления

- `--radius-pill: 90px` — пилюля: таб-бар и табы (концы полностью скруглены). Figma: `Radius/pill`.
- `--radius-card: 33px` — крупное: карточки, модалки, аватары, full-width кнопки дока.
- `--radius-medium: 20px` — среднее: инпуты, тосты, плашки, попапы, кнопки в модалках.
- `--radius-small: 10px` — мелкое: чипы-бейджи, скелетоны, маленькие пузырьки.
- Пилюли (теги группы мышц) — `999px`, отдельный осознанный случай.
- Круглые икон-кнопки — `50%` (см. «Икон-кнопки»).
- Аватары имеют свой радиус под рамку (≈29px на главной) — не трогать, завязано на `frames.js`.

### Вложенные радиусы (для переключателей/сегмент-контролов)

Концентрично: **внутренний радиус = внешний − padding контейнера.**
Пример: контейнер `--radius-medium` (20px) с `padding: 4px` → активная пилюля `borderRadius: 16px`.
Так скругления выглядят вложенными, а не случайными.

## Цвета — двухуровневые (примитивы → семантика)

В `index.css` цвета в 2 слоя:
- **Примитивы (палитра):** `--neutral-0/500/850/900/950`, `--green-400/500/700`, `--blue-400/500`,
  `--orange-500`, `--red-500`, `--purple-500`, `--pink-500`, `--yellow-500`. Сырые hex, без смысла.
- **Семантика (по назначению):** ссылается на примитив через `var()`, имена НЕ меняются —
  их используют компоненты. Менять смысловой цвет = переставить ссылку семантики на другой примитив,
  а не править hex в компоненте.

Семантика: базовые `--color-bg` `--color-card` `--color-card-hover` `--color-primary` (→ `--green-500`)
`--color-primary-dark` `--color-text` `--color-text-secondary`.
Категории: `--cat-strength` `--cat-cardio` `--cat-pool` `--cat-stretch` `--cat-recovery`.
Редкость рангов: `--rarity-common/rare/epic/legendary/mythic/godlike`.
Теги программ: `--tag-gym/home/outdoor`.
Полупрозрачные (альфа вшита, БЕЗ алиаса на примитив — примитивы сплошные):
`--color-surface-dim` (rgba(34,34,34,0.30) — стеклянный фон таб-бара и dim-кнопки),
`--color-border` (rgba(255,255,255,0.08) — тонкая обводка-волосок),
`--color-text-inactive` (rgba(255,255,255,0.5) — неактивные иконки/лейблы таб-бара).
**Конвенция записи цвета:** сплошной → hex (`#222222`); с прозрачностью → `rgba()` (точный
аналог «hex @ %» из Figma; 8-значный hex не даёт ровный процент). Не смешивать как попало.

### Таб-бар: нахлёст и тень
- **Нахлёст табов −5px** (в Figma = отрицательный item spacing −5 на контейнере). В коде CSS
  отрицательный gap нельзя → делаем `gap:0` + `marginLeft:-5px` у табов (кроме первого);
  активный таб `position:relative` + `zIndex` выше, чтобы пилюля перекрывала соседей.
- **Стекло/тень** = Effect Style (стиль эффекта), НЕ переменная (как Text Style). Таб-бар:
  `backdrop-filter: blur(8) saturate(180%)` (матовое стекло) + drop `0 8 40 / #000 12%` (iOS-док).
  Внутренний блик убрали (светился по всем краям). В Figma это Effect Style из 2 эффектов
  (Background blur + Drop shadow) — назвать **`effect/tabbar`** (не `shadow/...`, т.к. внутри ещё
  блюр). Liquid Glass (рефракция/дисперсия) в вебе не воспроизвести нативно — не тащим.
- **Обводка** бара — сплошная `1px var(--color-border)` (градиент-строук пробовали и откатили —
  визуально читался как заливка всего бара, не то).
- **Активный таб — стекло:** фон `--color-surface-active` (rgba(79,79,79,0.40) = #4F4F4F 40%;
  светлее трека, как активный сегмент в iOS/Telegram — НЕ темнее) + `backdrop-filter: blur(8)`
  на самом табе. Иконка активного «Тренировки» — бежевая `--color-icon-muscle` (#FADFBE).
  Неактивные иконки: fill `--color-text-inactive` (белый 50%) + opacity слоя 0.45 (≈22%, бледнее
  лейблов — так задумано). Зеркало код↔Figma — во фрейме-памятке в Figma (держать в синке).
Размытие (backdrop blur): `--blur-sm` (8px — таб-бар, dim-кнопка), `--blur-md` (12px —
серая neutral-кнопка/панель). В Figma — коллекция Blur (Number): `blur/sm`, `blur/md`.
Цвета мышечных групп — из `src/features/programs/colors.js` (`getMuscleGroupColors` → `{tag, accent}`,
есть fallback на серый). Цвета лиг — из `src/lib/leagues.js`.
Цвет тега программы — из `getProgramTagColor(tag, source)` в `registry.js` (единый источник, см. «Антидубль»).
Красный для ошибок/удаления: фон `rgba(232,69,69,0.16)`, рамка `rgba(232,69,69,0.5)`, текст `#FF6B6B`/`#E84545`.

### Заголовки групп мышц и закреплённый заголовок (экран дня)

Заголовки групп в потоке (`muscleHeader`) — **по центру** (как в конструкторе), в акцентном цвете
группы, **обычные** (ничего не гасим, отступы не трогаем). Когда листаешь вниз и заголовок уезжает
под карточку дня — появляется закреплённый заголовок текущей группы — **просто текст без своего фона**
(`WorkoutDay.jsx`, `groupPillRow` + `groupTabText`):
- Только текст в акцентном цвете группы. **Своего затемнения нет** — его даёт общий `stickyFade` выше,
  текст ложится поверх. `paddingTop` держит позицию.
- **Абсолютный, НЕ в потоке** (`position:absolute; top:100%` внутри sticky-шапки) — отступы списка не
  меняет, висит сразу под карточкой дня. Появляется/прячется по `pillGroup`. Смена/появление — мягко
  (`@keyframes groupPillIn`, ремаунт по `key={pillGroup}`).
- **Активная группа** = последняя секция, чей заголовок ушёл под карточку дня (верх заголовка ≤ низа
  `stickyHeader`). Считаем по `getBoundingClientRect` заголовков (`sectionHeaderRefs`) на scroll через
  rAF; вверху (ни один не ушёл) — `activeSection = -1`, заголовка нет. Стейт дёргаем только при смене группы.
- **Грабли (не повторять):** были версии с (1) full-bleed «фрост»-полосой + прозрачным `stickyHeader`
  + `headerOpaque` + гашением заголовков; (2) стеклянной пилюлей-чипом; (3) язычком-«челкой» с
  затемнением. Все откатили — оставили только текст поверх общего `stickyFade`. `stickyHeader` непрозрачный.

### Шапка дня (sticky-блок)

Шапка (`stickyHeader`) — `position:sticky; top:0`, непрозрачный фон `var(--color-bg)`,
`paddingTop: var(--tg-safe-top)`; контент скроллится ПОД ней и прячется. Внутри:
- `headerCard` — целиковый блок: верхняя строка (место слева, центр, крестик-отмена справа), крупная
  буква дня строго по центру, ниже — **чипы групп** (см. ниже), ещё ниже — счётчик/прогресс. Фон/строук
  как у карточки игрока на главной.
- **Центр верхней строки:** активна сессия → таймер; ДО старта → часы (`ClockIcon`) + оценка
  `≈ N мин` (`EST_MIN_PER_EXERCISE = 7` мин/упр: 3 подхода × ~2 мин отдых + переход; `slots × 7`).
- **Чипы групп дня** (`dayChips`) — по центру под буквой, каждый в цвете своей группы (`t.color`),
  белый текст, `opacity 0.7`. Идентичность дня; видны всегда. Ниже по центру — счётчик: до старта
  `N упражнений` (`pluralizeExercises`), в тренировке — `N/M` (без слова).
- **Прогресс дня — ЗАЛИВКА фона всей шапки** (`headerFill`), а не полоска/кнопка: светло-серый
  (`rgba(255,255,255,0.08)`) растёт слева по ширине `progressPct` за текстом, плавно
  (`transition width 0.55s ease-ios`). `headerCard` = `position:relative; overflow:hidden` (клип по
  скруглению), контент — в `headerCardInner` (`zIndex:1`). Только при активной сессии. Кнопка «Завершить»
  БЕЗ заливки/счётчика — просто `ЗАВЕРШИТЬ` (серая) → `✓ ЗАВЕРШИТЬ` (зелёная) на 100%; старт — `НАЧАТЬ`.
  Кнопки hug, без слова «ТРЕНИРОВКУ». Счётчик `N/M` в тренировке — крупнее/жирнее (как активный таймер,
  `dayCountActive` 15px/700).
- **Место (Зал/Дом)** можно менять и во время активной сессии (не залочено). Крестик-отмена — 44px
  хит-зона (см. «Закрытие модалки»).
- Под блоком — `stickyFade` + `stickySolid` + «челка» группы (см. выше).
- **Переключение дней** — свайпом влево/вправо (стрелки ← → и пейджер-точки убраны). Плюс **тап по букве**
  открывает `DayPicker` — стеклянную мини-модалку из ЦЕНТРА буквы (портал, `position:fixed` по её ректу):
  ряд дней A/B/C, тап по дню — переход, тап по фону — закрытие (уезжает в центр). **Раскраска ячеек:**
  запущенный день сессии (`sessionDay`) — **пульсирует** с обычного размера в ×1.5 (`.day-picker-pulse`),
  цвет зелёный если стоишь на нём, иначе серый (ты не на нём); текущий (не запущенный) день — белая буква
  на сером кружке (`cellCircle`, «ты тут»), зелёная если это ещё и рекомендованный; прочие — серые, без пульса.
  Буква В САМОЙ шапке — в цвете ПЕРВОЙ группы мышц дня (`dayGroupAccent`) когда просматриваешь «фокусный»
  день (сессия/рекомендованный), иначе серая.
- **Стики-заголовок группы** («челка», `groupPillRow`+`groupTabText`) пропадает, когда линия шапки
  проходит середину ПОСЛЕДНЕЙ карточки группы. Под карточкой дня — сплошная чёрная полоска
  (`stickySolid`, ~6px, всегда) в зазоре + ниже `stickyFade` (градиент+blur, опущен ~6px) для читаемости.

**Таймер тренировки (цвет/пульс/перегрузка/отмена).** Таймер строго по центру верхней строки
(`timerCenter` — `position:absolute; left:50%`). Цвет по порогам времени: старт → зелёный
(`--color-primary`), `≥TIMER_ORANGE_SEC` (1ч) → оранжевый `#F0883E`, `≥TIMER_RED_SEC` (1ч30) →
красный `#E84545`. На **смене** тира — пульс-увеличение (`@keyframes timerPulse`, ремаунт `span` по
`key={timerPulseKey}`; первый рендер не пульсирует). На 1ч30 — красноватый **поп-ап** «пора завершать» —
ПЛАВАЮЩИЙ поверх карточки (`position:absolute`, по центру над буквой, блюр-подложка, `@keyframes
overloadPopIn`), НЕ раздувает блок; висит пока не тапнешь ОК (один раз за сессию, `overloadShownRef`).
Справа — крестик «отменить тренировку» (только когда сессия активна) → модалка да/нет → `clearActiveWorkout`
+ `clearWorkoutProgress` БЕЗ `finishWorkout` (в историю не идёт). Сброс поп-апа/флага — в `handleStart`.

## Figma дизайн-система (зеркало токенов) — ДЕРЖАТЬ В СИНКЕ

Дмитрий дублирует дизайн-систему в Figma (файл **TRPG**, через локальный мост TalkToFigma).
Токены там — **Variables**, структура зеркалит `index.css` 1:1. Коллекции:
- **Primitives** (палитра, тип Color): группы `neutral/{0,500,850,900,950}`, `green/{400,500,700}`,
  `blue/{400,500}`, `orange/500`, `red/500`, `purple/500`, `pink/500`, `yellow/500`.
- **Semantic** (тип Color, ссылается на Primitives): группы `base/*`, `cat/*`, `rarity/*`, `tag/*`,
  плюс полупрозрачные с вшитой альфой (БЕЗ алиаса): `surface/dim` ↔ `--color-surface-dim`,
  `border/default` ↔ `--color-border`.
- **Radius** (Number): `card=33, medium=20, small=10`. **Size** (Number): `tabbar-height=72, tabbar-bottom=16`.

**Соответствие имён** (детерминированное): Figma `green/500` ↔ CSS `--green-500`;
`base/primary` ↔ `--color-primary`; `cat/strength` ↔ `--cat-strength`; `Radius card` ↔ `--radius-card`.
Группа через `/` в Figma = дефис-префикс в CSS.

**ПРАВИЛО СИНКА (обязательно):** любое изменение токена в `index.css` (новый цвет, смена hex
примитива, перепривязка семантики, новый радиус) → **в конце ответа напомнить Дмитрию продублировать
это в Figma**, с точной инструкцией: коллекция → имя переменной → новое значение/ссылка.
Пример напоминания:
> 🎨 Синхронизируй Figma: Primitives → `green/500` → значение `#8FC04A` (было `#9ED153`).

И наоборот: Дмитрий меняет Variables в Figma → присылает (скрин/ссылку на узел) → переносим в код.
Читать переменные через мост: значения видны на узле, но привязку «это Variable» он не отдаёт —
сверять по скрину панели Variables или по залитому узлу.

## Шрифты

- `--font-geist` — заголовки/названия (название упражнения в карточке).
- `--font-manrope` — основной текст, body (дефолт).
- `--font-display` — акцентный шрифт (Geist) для игровых элементов: ранг, XP, медали, серии,
  секции-лейблы, крупные числа. Бывш. пиксельный Tiny5.
  **Жирность по иерархии** (Geist загружен 400–800): крупные числа/тайтлы ≥28px → `800`,
  средние акценты 15–27px → `700`, мелкие капс-лейблы и теги ≤14px → `600`. Всегда задавать
  явный `fontWeight` (иначе Geist падает в 400 и выглядит хилым).

## Press-эффекты (классы из index.css)

Любой нажимаемый элемент должен иметь press-фидбек. Классы:
- `.press-tile` — scale 0.97, для плиток/кнопок (в т.ч. full-width кнопки дока — не забывать вешать).
- **«Растущее» нажатие с отменой** (крестик-пузырёк под медиа-модалкой, `ExerciseActionMenu`) — НЕ через
  CSS `:active` (на тач указатель неявно захватывается кнопкой → `:active`/`leave` не гаснут при уводе
  пальца, и `click` всё равно закрывает). Делать pointer-событиями: `pointerdown` — arm + scale 1.14;
  `pointermove` — хит-тест по `getBoundingClientRect`, палец вне кнопки → disarm + вернуть размер;
  `pointerup` — закрыть только если ещё armed; `onClick` — лишь `stopPropagation` (не закрывает).
- `.press-exercise-card` — scale 0.98, для крупных карточек.
- `.tg-row` — без scale, фоновая подсветка при нажатии (строки-списки как в Telegram).
- `.press-danger` — для кнопок «закрыть/удалить»: при нажатии красная заливка `rgba(232,69,69,0.18)`
  + красный значок. `!important` (перебивает инлайновый фон кнопки). Вешается рядом с `.press-tile`
  (scale сохраняется). Если красный фон не нужен (напр. крестик-очистка поиска) — только `.press-tile`.
- `.press-grow` — для инпутов/полей: на `:focus`/`:active` лёгкое `scale(1.02)`, видно что тапнул.

## Икон-кнопки (круглые)

Кнопки-иконки одного семейства (добавить `+` / убрать `✓` / удалить `✕` / закрыть `✕`) держим
**одного размера и формы** во всех экранах, чтобы читались как одна кнопка:
- `36×36`, `borderRadius: 50%`, глиф `fontSize: 18px / fontWeight: 700`.
- Центрировать символ через `display:flex` + `alignItems/justifyContent: center` + `lineHeight: 1`.
  Текстовые глифы (`✕`/`+`/`✓`) сидят не по оптическому центру — добавлять доводчик `paddingBottom: 1–2px`.
  Надёжнее рисовать глиф как inline-SVG (как `GripIcon` в конструкторе) — тогда центр геометрически точный
  без подбора пикселей.
- **Неактив — нейтральный**: серый фон `rgba(255,255,255,0.06)` + `--color-text-secondary`.
  Акцентный цвет (зелёный) — только для активного/подтверждённого (выбранная галочка `✓`).
  Не заливать неактивную кнопку ярким `--color-primary`.
Эталон — кнопка `+/✓` в `ExercisePicker`; к ней приведены крестик удаления в `ProgramConstructor`
и крестик-очистка поиска (последний крупнее — 44px, т.к. часть поля поиска).

## Карточка упражнения

Единый вид строки-упражнения. Базовые размеры одинаковы на всех трёх экранах — различается только
правая зона и левый край.

**База (эталон — экран смены `SwapExercise.jsx`):**
- Контейнер: `flex; alignItems:center; gap:12px; padding:12px; minHeight:90px;`
  `background: var(--color-card); borderRadius: var(--radius-card)`.
- Превью: `64×64`, `var(--radius-medium)`, белый фон, `objectFit:cover`;
  плейсхолдер-эмодзи `fontSize:28px; opacity:0.4`.
- Контент: колонка, `gap:5px`.
- Имя: `var(--font-geist)` `13px / 700 / lineHeight:16px`, перенос разрешён (без ellipsis).
- Тег: `10px / 700`, `borderRadius:999px`, `letterSpacing:0.2px; lineHeight:13px`. **Один тег —
  подгруппа** (`SUB_GROUP_LABELS`, фолбэк на имя группы) в цвете **основной** группы
  (`getMuscleGroupColors().tag`, белый текст), **`opacity: 0.7`** (спокойный, приглушённый вид). Имя
  группы (Спина/Грудь) не дублируем — оно в заголовке секции. **Правило единое ВЕЗДЕ, где есть тег
  упражнения:** `ExerciseCard`, `ExercisePicker` (строки списка), `SwapExercise`, `ProgramConstructor`,
  а также **`ExerciseActionMenu`** (долгий тап) и **`ExerciseHeaderCard`** (техника упражнения) — в этих
  двух РАНЬШЕ было два чипа (группа+подгруппа), теперь ТОЛЬКО подгруппа в цвете группы, `opacity 0.7`.
  Чипы групп дня в шапке (`WorkoutDay.dayChips`) — тоже `opacity 0.7`.
- **Фильтр-чипы в пикере (`ExercisePicker`) — НЕ теги, свои правила, БЕЗ opacity (яркие):** верхний ряд
  групп — активный чип залит цветом СВОЕЙ группы (`getMuscleGroupColors(group).tag`, белый текст);
  ряд подгрупп активной группы — активная подгруппа тоже заливается **цветом группы** (`.tag`), НЕ общим
  зелёным. Неактивные — серый фон + `--color-text-secondary`.

**Группировка по группе мышц.** И на дне тренировки (`WorkoutDay`), и в конструкторе упражнения идут
секциями по основной группе (подряд идущие одной группы = секция), над секцией — заголовок группы
в её **акцентном** цвете (`getMuscleGroupColors().accent`): `var(--font-display)` `13px / 600`,
`letterSpacing:2px`. Заголовок **по центру** и в конструкторе, и на дне (на дне поверх — закреплённая
стеклянная пилюля текущей группы, см. «Заголовки групп мышц и закреплённая пилюля»).

**Вариант в пикере (`ExercisePicker.jsx`):** та же карточка, справа круглая кнопка `+/✓` (см. «Икон-кнопки»).

**Вариант в конструкторе (`ProgramConstructor.jsx`):** та же карточка + drag-ручка вынесена **за** карточку
в левый жёлоб (чтобы превью стояло от края, как в эталоне), справа крестик удаления:
- Внешняя обёртка `exRowWrap`: `flex; alignItems:center; gap:6px`.
- Внутри — ручка (`width:28px; alignSelf:stretch; touchAction:none; cursor:grab`, глиф — inline-SVG)
  + сама карточка (`flex:1`).
- Перетаскивание — **простой transform-сдвиг** на pointer-событиях, без сторонних библиотек. Сознательно
  откатились сюда: версии с живой сортировкой (клон + FLIP/WAAPI + своп) дребезжали/лагали, Дмитрий
  попросил вернуть эту — плавную. Как работает:
  - Захваченная строка сдвигается `transform: translateY(dy) scale(1.03)` (за пальцем), соседи между
    стартом и целью разъезжаются на `±stride` через CSS-`transition` (`rowDragStyle`). `stride =
    offsetHeight + 10` (высота строки + gap списка). Цель = `startIndex + Math.round(dy/stride)`.
    Порядок применяется **на отпускании** (`moveItem`), не на лету. Строки держим по `rowRefs[idx]`.
  - `dy` включает **сдвиг скролла**: `(pointerY−startY) + (scrollY−startScrollY)`. Это позволило вернуть
    **авто-скролл** (`startAutoScroll`, rAF): палец у верх/низ края (`EDGE_TOP_PX`/`EDGE_BOTTOM_PX`) →
    `window.scrollBy`, и `applyDrag` пересчитывает — карточка остаётся у пальца, цель сдвигается. Авто-
    скролл тут НЕ дёргается (нет `getBoundingClientRect`/живого реордера — только арифметика + transition).
  - Группировка в это НЕ вмешивается: заголовки секций при drag статичны. Через границу группы мелкая
    неточность («заголовок чуть мозги делает») — мирятся ради плавности; внутри группы пиксель-в-пиксель.
  - Список рендерится секциями (`daySections` → `daySection` + `sectionRows`), gap внутри секции `10px`
    (= stride), между секциями `20px`. Без клонов/порталов/WAAPI — отсюда и плавность.
- **Кнопка «Добавить упражнения» — в потоке** (последним элементом списка, под последним упражнением /
  под подсказкой «Пусто…» у пустого дня), а НЕ прибита к низу. Внизу в доке остаётся только
  «Сохранить программу». При лимите (10/10) тап даёт тот же тост. Низ страницы (`paddingBottom`) держит
  место под док, чтобы кнопка прокручивалась выше него.

## Активное состояние без заливки

Активность/выбор показываем акцентом (цвет + размер), а не яркой заливкой фона.
Фон элементов остаётся нейтральным серым, активный отличается цветом `--color-primary` и/или
чуть большим размером. Примеры: выбор кол-ва дней в конструкторе (активная цифра зелёная и крупнее,
фон всех боксов одинаковый серый); кнопка `✓` в пикере (зелёный значок, а не зелёная заливка).

## Места тренировки (Зал/Дом/Улица) и пилюли-вкладки

- Места — единый источник `PLACE_META`/`getPlaceMeta`/`getProgramPlaces` в `registry.js`:
  `gym`=Зал 🏋️, `home`=Дом 🏠, `outdoor`=Улица 🌳 (эмодзи пока временные), цвет иконки =
  `--tag-gym/home/outdoor`. Порядок фиксированный: зал → дом → улица. `getProgramPlaces(program)`
  → какие места заполнены (из `data.locations`; для Сплита по тегу 'зал').
- У кастомной программы тег «своя» убран — вместо него показываются места из `data.locations`.
- **PlaceSwitcher** (`components/PlaceSwitcher.jsx`) — переключатель места в шапке экрана дня (тапается,
  меняет место). На КАРТОЧКАХ (главная/избранное/раздел, через `FavCardBody`) — проп **`tag`**: статичный
  тег выбранного места (вид активного сегмента без контейнера/обводки), НЕ тапается.
  **Вид = сегмент-контрол мест из конструктора** (контейнер-пилюля `--color-surface-dim`+`--color-border`
  +`blur(--blur-sm)`, `--radius-pill`; активная позиция залита `--color-surface-active` и покрашена
  **цветом места** `getPlaceMeta(loc).color`, неактивные — `--color-text-inactive`; нахлёст −5 + zIndex
  активного). Компактнее конструктора (item `minHeight 26`, font 12, icon 16). Свёрнуто — одна пилюля
  (активное место); тап раскрывает остальные справа (серым), тап по другому — выбор+свёртка. Выбор —
  через хук `useProgramPlace(slug, places)`
  (`lib/program-place.js`, ключ `program-place:<slug>`, CloudStorage, кросс-девайс), общий для карточек
  и тренировки. По умолчанию неконтролируемый; на экране дня — controlled (`value`+`onChange`), там же
  смена места перезагружает день (`getWorkoutDay(slug, day, place)` грузит набор из `data.locations`).
  Одно место — статичный тег. `getProgramPlaces(program).length===0` (заплыв) → не рендерится, там
  обычные `prog.tags`. Глушит всплытие (тап ≠ тап по карточке).
  Прогресс дня раздельный по месту (ключ `workout-progress:{slug}:{place}:{day}`). Известное
  ограничение (на потом): свапы дня (замены упражнений) привязаны к `(program, day)` без места —
  между местами одного дня могут пересекаться.
- **Сегмент-контролы конструктора = мини-таб-бар** (`segGroup`+`segItem`+`segItemActive`). Контейнер
  `segGroup` повторяет нижний таб-бар: фон `--color-surface-dim`, обводка `--color-border`,
  `blur(--blur-sm) saturate(180%)`, паддинг 4, тень. Табы `segItem` — прозрачные (как неактивный
  таб) + текст `--color-text-inactive`; активный `segItemActive` — залит `--color-surface-active` +
  блюр + акцентный зелёный текст. Нахлёст табов −5 (`marginLeft` инлайн + `zIndex` активного выше),
  `alignSelf:stretch`. Увеличивается ТОЛЬКО текст (не сам таб), без нижней полоски. Всего 2 заливки —
  как в таб-баре. Дни-в-программе и дни A/B/C — контейнер на всю ширину (`flex:1`).
- **Место (Зал/Дом/Улица) — «таб-бар на одну активную позицию»**: контейнер `segGroup` (`width:auto`,
  табы `flex:0 0 auto`) обнимает только места, которые `placeFilled(loc) || (active && placeTouched)`;
  активная позиция залита. Пустые невыбранные места — снаружи контейнера голым текстом (`placeBare`,
  `rgba(255,255,255,0.3)`). До первого тапа по месту (`placeTouched=false` у новой программы)
  контейнера нет — все места голым текстом. Иконки мест (UiIcon) 21px. Активная позиция: текст+иконка
  красятся **цветом самого места** (`getPlaceMeta(loc).color` → `--tag-gym` оранж / `--tag-home` синий /
  `--tag-outdoor` зелёный), НЕ общим зелёным `--color-primary`; неактивные — `--color-text-inactive`.
  (Дни A/B/C и кол-во дней остаются зелёными — цвет места только у мест.)

## Секции-лейблы и отступы (правило близости)

Заголовок секции — стиль `secLabel`: `var(--font-tiny5)`, `12px`, `--color-text-secondary`,
`letter-spacing: 1.5px`, `margin-bottom: 10px` (лейбл липнет к своему контенту).
Между секциями — `20px` (секции разнесены). Так связанное стоит близко, разное — дальше.
Пример (конструктор): НАЗВАНИЕ → инпут, ДНЕЙ В ПРОГРАММЕ → боксы, ДНИ → вкладки A/B/C, УПРАЖНЕНИЯ → список.

## Кнопка дока (нижнее основное действие)

Full-width кнопка внизу экрана («Добавить упражнения» в пикере/конструкторе):
`padding: 18px`, `borderRadius: var(--radius-card)`, dashed-рамка `1.5px rgba(255,255,255,0.18)`,
фон `rgba(34,34,34,0.55)` + `backdrop-filter: blur(12px)`, текст `13px / 700`. Обязательно `.press-tile`.

Прятать кнопку при открытой клавиатуре (visualViewport, порог 150px), чтобы не липла к клавишам.

## ActionButton — общий компонент кнопки-действия (`components/ActionButton.jsx`)

Единая кнопка для всех основных «прибитых» действий. Рамка СПЛОШНАЯ (пунктир оставлен только
кнопке «Добавить упражнение» в конструкторе/пикере — там постоянно редактируем).
`<ActionButton variant disabled hug onClick style>…</ActionButton>`. Варианты:
- **dim** (всегда при `disabled`) — прозрачный фон `rgba(34,34,34,0.30)` + `blur(8px)`, текст тусклый,
  рамка `1.5px solid rgba(255,255,255,0.12)`. Это «неактивное» — не городить `opacity`.
- **neutral** — серая полупрозрачная `rgba(34,34,34,0.55)` + `blur(12px)`, рамка `…0.20`.
- **accent** — сплошная зелёная заливка `--color-primary` + чёрный текст `#0D0C0C`, рамка
  `1.5px solid --color-primary-dark` (как кнопка в инфо-попапе рейтинга). Это «готово/доступно».
**Высота — 55px, скругление `--radius-pill` (полная пилюля)**: `ActionButton`, а также док-кнопки
«Добавить упражнения» (конструктор/пикер) — все кнопки действий одной высоты 55 и пилюльной формы.
(Таб-бар отдельно — 63.) На 55px радиус 33 и 90 визуально одинаковы (стадион), но в коде/Figma — 90.
Ширина: по умолчанию full-width; проп **`hug`** — по размеру контента (скругление облегает
текст), контейнер тогда центрирует (`display:flex; justify-content:center` или `alignItems:center`
у колонки-дока). Высоту/типографику переопределять через `style` (иконку слева — `style={{gap:8}}`
+ `<UiIcon color="#0D0C0C">`). **Все основные кнопки — `hug`** (так выглядит лучше): «Завершить»,
«Сменить» (SwapExercise), «Сохранить программу» и «Добавить упражнения» (конструктор/пикер; последняя —
пунктир, не ActionButton, но тоже hug), «Добавить друга»/«Пригласить друга» (Friends, accent). Любая
нажимаемая кнопка-действие — через него, а не инлайн. (Серые кнопки дисмисса инфо-попапов — `ModalButton`.)
- **Проп `progress` (0–100)** — заливка-прогресс за текстом (светло-серым, растёт по мере действия;
  обрезается по скруглению). Общий, но кнопка «Завершить» его СЕЙЧАС НЕ использует — прогресс дня
  показывает заливка шапки (`headerFill`, см. «Главная — шапка»). Кнопка «Завершить» — обычная hug:
  `neutral` до 100%, `accent` (зелёная, с ✓) на 100%.

### Кнопка подстраховки (`BackupButton`)

Единый компонент для ВСЕХ кнопок подстраховки: «Подстраховать» в модалке игрока
(`PlayerProfileModal`) и «Подстраховать всех» на странице друзей (`BackupAllButton`). Один вид:
**серая капсула** (`rgba(255,255,255,0.06)` + обводка `…0.10`, НЕ зелёный accent), высота ~44px
(на ~15–20% ниже `ActionButton`), текст **обычным регистром** (без `text-transform`), бицепс рядом
с подписью. НЕ через `ActionButton`. Пропсы: `onClick disabled dim pulse flyer children`.
`dim` — приглушённый текст для статусов (отправка / уже подстрахован / лимит). `flyer={{bonus,key}}`
— улетающий бицепс с `+бонусом` (массовая/разовая). Лимит-счётчик ОТДЕЛЬНОЙ надписью не показываем:
сообщение о лимите — только ВНУТРИ кнопки и только когда лимит реально достигнут
(`Лимит N/N — возвращайся завтра`).

## Закрытие модалки — крестик + тап по фону

Модалки закрываем серым **крестиком** (а не кнопкой «Закрыть») ПЛЮС тап по фону-оверлею
(`overlay onClick={onClose}`, контент — `stopPropagation`). Крестик — inline-SVG `CrossIcon` (две линии
`currentColor`, скруглённые концы), серый кружок `background: rgba(255,255,255,0.06–0.08)`,
`borderRadius:50%`, цвет `--color-text-secondary`. Две раскладки:
- **В правом верхнем углу** (обычные модалки): кружок ~30px, `position:absolute; top/right ~14px`,
  контейнер `position:relative`. Эталон — `PlayerProfileModal`.
- **Крупный «пузырёк» ПОД модалкой по центру** — для медиа-модалок (`ExerciseActionMenu`: видео+карточка
  +заметка): оверлей `flex-direction:column`, под модалкой кружок **46px** (хит-зона **56px**),
  `marginTop 14px`, отступ низа под `env(safe-area-inset-bottom)`. Низ удобнее для большого пальца.
- **Хит-зона любого крестика/икон-кнопки — ≥44px** (зал, потные пальцы): прозрачная кнопка 44–56px
  вокруг видимого кружка 30–46px. Так же сделан крестик «отменить тренировку» в шапке дня (`cancelBtn`
  44px + `cancelBtnInner` 30px).

**Грабля (авто-рост textarea в модалке):** не звать resize в ref-колбэке (`ref={el => autoGrow(el)}`) —
он срабатывает на КАЖДЫЙ ререндер, и при фокусе/клавиатуре `height='auto'` сбрасывался многократно →
каретка дребезжала. Правильно — `useLayoutEffect` на `[editingNote, draft]` (до отрисовки), а ref-колбэк
только присваивает `ref.current` (см. `ExerciseActionMenu`).

## ModalButton — серая кнопка для модалок/диалогов (`components/ModalButton.jsx`)

Нативный серый стиль (как кнопки iOS-алерта «Понятно/ОК/Отмена»), НЕ акцентный — для дисмисса
инфо-попапов и нейтральных выборов (в отличие от зелёной ActionButton accent для основных действий).
`borderRadius: --radius-pill` (полная пилюля), фон и подсветка нажатия — в классе `.modal-btn`
(index.css): база `rgba(255,255,255,0.13)`, `:active` ярче `0.22`, БЕЗ scale (как `.tg-row`).
Фон держим в CSS (не инлайн), иначе `:active` его не перебьёт. По умолчанию full-width; в ряд (две
кнопки) — обернуть в `display:flex; gap:10` и каждой `style={{flex:1}}`. Применяется: «ПОНЯТНО» в
инфо-попапах (рейтинг `RulesModal`, разделы `CategoryInfoModal`), пара [Не сохранять][Сохранить] в
модалке выхода из конструктора. Кнопки «Понятно/ОК/Отмена» — больше НЕ ярко-зелёные.

## Лимит-индикатор (красный, при недопустимом действии)

Красное сообщение про лимит + шейк + `haptic.error()`. Два размещения под разный UX:

- **Пузырёк у элемента** — когда действие построчное (пикер: тап `+` на затемнённой строке сверх лимита).
  Пузырёк рисуется в строке слева от кнопки, ровно напротив неё: обёртка кнопки `position:relative`,
  пузырёк `position:absolute; right: calc(100% + 10px); top:0; bottom:0; display:flex; alignItems:center`
  (высота = высота кнопки → центр совпадает). Текст короткий («Лимит 10/10»), `radius-small`, blur-фон.
  Затемнение строки вешать на содержимое (превью/контент/кнопку), НЕ на всю строку — иначе пузырёк
  тоже приглушится; он должен оставаться ярким.
- **Центр-попап у дока** — когда кнопка одна общая (конструктор: тап «Добавить» при забитом дне).
  `position:fixed`, по центру над доком, `maxWidth ~200px` → текст в 2 строки, `radius-medium`, blur-фон.

Общее: хранить не булев флаг, а id/нонс, чтобы шейк перезапускался на каждый тап (`key={nonce}`).

## Анимации

- `--ease-ios: cubic-bezier(0.32, 0.72, 0, 1)` — основной easing.
- `--duration-page: 280ms` — переходы страниц.
- Press: `--press-scale` `--press-duration` `--press-ease`.
- Уважать `@media (prefers-reduced-motion: reduce)`.
- Переиспользуемые keyframes держать в `index.css`, не плодить копии в компонентах.
- **Без постоянного фона из частиц.** Раньше был `ParticlesBg` (зелёные частицы летели снизу
  вверх на всех экранах) — убран как отвлекающий. Остались только событийные всплески в
  `ParticlesBg.jsx`: `spawnFireSparks` (искры серии), `spawnBurst`, `spawnQuestBurst` — их и
  использовать для фидбэка «при получении». Постоянный амбиентный фон не возвращать.

### Шейк ошибки (`.shake-error` + `@keyframes errorShake`)

Влево → толчок вправо (`-7/+6/-4/+3px`, 0.4s). Для тостов/попапов/полей при недопустимом действии,
в паре с `haptic.error()`. Перезапуск на каждый тап — через `key={nonce}` (ремаунт).
**Грабля:** инлайновый `animation` на элементе перебивает класс-анимацию (инлайн сильнее) — не вешать
`animation` инлайном на элемент с `.shake-error`. И `.shake-error` НЕ вешать на элемент, у которого
`transform` уже занят (напр. центрирование `translateX(-50%)`) — шейк перебьёт центрирование; в этом
случае заворачивать в обёртку: центрирование на обёртке (`justify-content`), шейк на внутреннем.

### Рамки рангов на аватарах (`RankFrame`)

Единый компонент `components/RankFrame.jsx` — аватар в рамке ранга. Использовать ВЕЗДЕ, где
рисуется аватар с рамкой: профиль (`ProfileHeader`), списки (`FriendRow`, `LeaderboardRow`),
модалка игрока (через `ProfileHeader`). Контент (фото/буква) передаётся через `children`.
Индекс ранга — `rankIndexFromMuscles(total_muscles)` из `lib/frames.js` (формула `/900`,
как в БД). Рамки 8/9/10 анимированные (`frame-titan`/`frame-legend`/`frame-immortal` в `index.css`),
0–7 — статичная полоска цвета ранга (`glow` добавляет мягкое свечение, для профиля).

**Грабли (почему сделан единый компонент):**
- **Толщина рамки постоянна, пульсирует только свечение СНАРУЖИ** (`box-shadow`: сияние + кольцо
  `0 0 0 Npx`). Раньше `titanSwell`/`immAura` анимировали `border-width` (3↔4px) — из-за
  `box-sizing: border-box` это сжимало/растягивало фото («дышало»). Не анимировать `border-width`.
- **Цвет обводки задавать inline (`frame.color`)**, а не только классом: inline-shorthand
  `border: 'Npx solid'` без цвета сбрасывает border-color в `currentColor` (белый) и перебивает
  класс → была белая рамка вместо красной. В `RankFrame` ставим `border` ДО `borderColor`.
- **Легенда (9):** conic-обводка живёт в CSS — НЕ задавать свой `background`/`border` inline
  (перебьют conic). `RankFrame` для `frame-legend` удаляет inline-фон.
- `box-shadow`-свечение НЕ обрезается своим `overflow: hidden` (чужой `overflow` у предка —
  может). Слой пепла Бессмертного (`.imm-ash`) — внутри рамки.

## Пользовательский контент

Контент, который вводит юзер (название своей программы), показываем **как есть** — не нормализовать
регистр. `toLowerCase()`-нормализация («Первая заглавная, остальные строчные») — только для встроенного
контента. Развилка по `source === 'custom'`. Кастомной программе — свой акцент: тег зелёный (`--color-primary`).
Это правило действует **везде**, где выводится имя программы: карточки (Home, Category), карусель
избранного И **история** (`describeWorkout` в `utils/history.js`) — частая грабля: история гнала всё
через `titleCase` и ломала регистр кастомных имён.

## Заголовок экрана (ScreenTitle) — единый для всех экранов

Заголовок любого экрана — компонент **`ScreenTitle`** (`components/ScreenTitle.jsx`): стиль навбара
iOS/Telegram — `--font-manrope`, **18px, вес 600, БЕЗ капса** (sentence case), по центру, приглушённый
белый `rgba(255,255,255,0.9)`. Размер/вес/цвет правятся в одном месте (компоненте) — сразу на всех экранах.
Применён везде (Тренировки, Профиль, Друзья, Рейтинг, Разделы, Избранное, История, Награды, Настройки,
Заплыв, Сменить упражнение, Своя программа…). Новый экран → заголовок через `ScreenTitle`.

**Это ФИКСИРОВАННЫЙ навбар, НЕ в потоке.** Рендерится `createPortal` в `document.body` как
`position:fixed`-полоса по центру системных кнопок Telegram (Назад / …) — заголовок встаёт в одну линию
с ними. Позиция из переменных `--tg-nav-top` (=`safeAreaInset.top`) и `--tg-nav-height`
(=`contentSafeAreaInset.top`), их задаёт `bindSafeArea()` (фолбэк в `index.css`). Портал ОБЯЗАТЕЛЕН:
`.page-enter/.page-fade` оставляют `transform` (fill-mode `both`) → создавали бы контейнер для `fixed` и
сбивали позицию. `zIndex:90` (ниже OfflineBanner 9998 — при оффлайне пилюля перекрывает заголовок, это ок).
Из потока заголовок ВЫНЕСЕН → контент экрана начинается ровно на 16px ниже кнопок (`paddingTop:
var(--tg-safe-top)` у `.page`), БЕЗ строки под заголовок и БЕЗ своего верхнего margin у первого элемента.
Иконки/сабтайтлы/боковые кнопки (кубок, ⓘ, шестерёнка, сердечко) идут первой строкой контента, НЕ в
строке заголовка. `style`-проп компонент больше не принимает (позиция фиксированная).

## Заголовок экрана раздела (Category)

Шапка экрана категории — вертикальная колонка по центру: иконка раздела (`UiIcon`, 40px, в цвете
категории) сверху → белый заголовок (`--font-display`, 26px, 800, `--color-text`) → серый подзаголовок.
Цвет уходит на иконку, текст белый. Подзаголовок — динамичный счётчик программ (см. ниже), не статика.
Справа вверху шапки — серая ⓘ-кнопка инфо (см. «Инфо-поповер»).

## Динамичный счётчик программ

Подпись количества программ в разделе — всегда из реестра, не хардкод: `programCountLabel(categoryId)`
из `registry.js` (`getProgramsByCategory().length` + склонение `pluralPrograms`: 1 программа /
2 программы / 5 программ). Используется на главной (подзаголовок раздела) и в шапке Category.
Меняется сам при создании/удалении своих программ и добавлении программ друзей.

## Инфо-поповер (ⓘ)

Объяснялки «прочитал один раз» (правила рейтинга, польза раздела) — не текстом в макете (отвлекает),
а по серой ⓘ-кнопке (`UiIcon name="info"`, цвет `--color-text-secondary`; в рейтинге была синей,
дефолт теперь серый). Поповер — портал (`createPortal` в `document.body`), оверлей
`rgba(13,12,12,0.85)` + `blur(10px)`, панель `maxWidth 340px`, `--radius-card`, появление через
keyframes (overlay fade + panel scale). Эталоны: `RulesModal` в `Leaderboard.jsx`,
`CategoryInfoModal` в `Category.jsx`. Тап по фону закрывает, кнопка «ПОНЯТНО» снизу.

## Главная — шапка, разделы, дневной буст

- **Строка недели** под заголовком «Тренировки» — серая пилюля по центру (`weekStatus`): огонёк
  `StreakFlame streak={N}` (тот же, что в профиле) + «×N» + `pluralizeWorkouts`. `N` = `weekly_streak`
  этой недели (`getCurrentUser().weekly_streak`, сверка `weekly_streak_week === getCurrentWeekKey()`).
  При `N=0` — зовущая фраза «Начни тренировку — стань лучшей версией себя». Обновляется на событиях
  `USER_CHANGED`/`USER_READY`.
- **ИЗБРАННОЕ** — заголовок-label (НЕ кликается) + «Все ›» справа (`seeAllBtn` → `/favorites`).
  Убрана «двойная дверь» (раньше и заголовок, и карточки вели в одно место). Само избранное — карусель,
  не сворачивается.
- **Секции РАЗДЕЛЫ / ДНЕВНОЙ БУСТ** сворачиваются тапом по заголовку (серый шеврон справа,
  `rotate(180deg)` при раскрытии). `SectionToggle`+`HomeChevron` в `Home.jsx`, контент по `!collapsed`.
  Состояние — кросс-девайс CloudStorage (`home-sections-collapsed`, `{ sections, boost }`).
  **РАЗДЕЛЫ по умолчанию раскрыты**, **ДНЕВНОЙ БУСТ по умолчанию свёрнут** (`boost !== false`).
- **Дневной буст на главной — компактный виджет:** в заголовке живая сводка
  `getDailyBoostSummarySync()` → `ДНЕВНОЙ БУСТ N/3 · +N💪` (или ` ✓`), развёрнутый список — `DailyQuests`
  (тот же, что в Профиле). В блоке «день пройден» строку «+60 получено» убрали — осталась «✔ День пройден».
- **Разделы:** `CategoryList` показывает все 4 (Силовая, Плавание, Кардио, Растяжка); Кардио/Растяжка —
  с тегом «Скоро», внутри по 1 плейсхолдеру (`PLACEHOLDER_PROGRAMS` в `Category.jsx`). Строки — с
  приглушённо-серой стрелкой `›` (`rgba(255,255,255,0.25)`). Настройка «Отображение на главной»
  (home-layout) удалена целиком.

**Карточка программы (`ProgramCard`/`FavCardBody`):**
- **Место (Зал/Дом)** на карточке — **статичный тег** (`PlaceSwitcher tag`): показывает выбранное место
  (вид активного сегмента, но БЕЗ контейнера/обводки), НЕ тапается. Менять место можно ТОЛЬКО внутри
  тренировки (там `PlaceSwitcher` обычный). Бейдж `▶ Продолжить` — УБРАН.
- **Не активна:** тег места → ряд дней A/B/C (`17px`, подсветка рекомендованного) → `Последняя · N`
  (`showLast`+`lastLabel`; нет даты — пусто).
- **Активна тренировка:** крупная буква активного дня (`22px`, `dayLetterActive`) + `N/M` + время — в
  одну линию (`activeRow`); ниже — тег места. Отдельной статус-полоски НЕТ.
- **Прогресс дня — заливка ВСЕЙ карточки** (`ProgramCard.cardFill`, светло-серый `rgba(255,255,255,0.08)`,
  растёт по `activeDone/activeTotal`, `transition 0.55s`, `overflow:hidden` клип по скруглению; контент —
  `zIndex:1`). Как в шапке дня (`headerFill`).
- **Цвет буквы дня** (подсвеченного/активного) = акцент ПЕРВОЙ группы мышц дня (`getMuscleGroupColors(
  prog.data.days[d][0].muscle_group).accent`), не общий зелёный; невыбранные — серые. Та же логика у
  крупной буквы в шапке дня (`WorkoutDay`, по `dayTags[0]`).

**Огонёк-серия ×N — цвет по состоянию:** `x0` серый (`rgba(255,255,255,0.4)`, как негорящий огонёк),
`≥1` оранжевый `#FF8C42`. Применяется везде, где рисуется `StreakFlame` + «×N» (главная `weekMult`,
`ProfileHeader.statCount`/`streakCount` — профиль, рейтинг, модалка игрока).

## Telegram Mini App

- **Хаптика** — через `haptic` из `src/lib/telegram.js`. Методы: `light/medium/heavy/soft/rigid`
  (impact), `success/warning/error` (notification), `selection`. Раскладка:
  `haptic.light()` — обычный тап, `haptic.medium()` — long-press,
  `haptic.success()` — сохранение/успех, `haptic.error()` — недопустимое действие (в паре с `.shake-error`),
  `haptic.selection()` — микро-интеракции (фокус поиска, очистка поля, переключение чипов/сегментов,
  шаг перетаскивания). Любой интерактив — с хаптикой.
- **Safe area** — `--tg-safe-top` (фолбэк 116px, перезаписывается реальным из `bindSafeArea()` =
  `sys+ui+8`). Также `bindSafeArea` задаёт `--tg-nav-top`=`sys` (вырез) и `--tg-nav-height`=`ui`
  (полоса кнопок Telegram) — по ним позиционируется фиксированный `ScreenTitle` (см. «Заголовок экрана»).
  Таб-бар: `--tabbar-height` **63px** + `--tabbar-bottom` 16px. Контент страниц — класс `.page`.
- **Краевой скрим** — `.app::before` (верх, под навбаром-заголовком) и `.app::after` (низ, над
  таб-баром): один рецепт градиент `var(--color-bg)→transparent` + `blur(3px)` + маска (гаснет без
  линии), `position:fixed`, одинаковая высота `var(--tg-safe-top)`, `z:90`. Блюрит контент за собой
  при скролле; заголовок навбара (`z:95`) и таб-бар (`z:100`) выше — остаются резкими. `.app::after`
  гасится на конструкторе (`body.hide-app-scrim`) и при клавиатуре; `.app::before` — всегда.
  Верхний `.app::before` чуть ниже (`height: calc(safe-top + 16px)`, маска до 65%) — текст под
  заголовком при скролле мутный/тёмный, сам заголовок (`z:95`) резкий.
- **`.dock-scrim`** — тот же рецепт для экранов со своими прибитыми доками, где `.app::after`
  скрыт (день/конструктор/пикер/смена). Это отдельный `<div className="dock-scrim" />` ПЕРВЫМ
  ребёнком дока (`position:absolute; inset:0; z-index:-1`) — блюрит контент за доком, но маска НЕ
  режет кнопки (они выше по слою). У дока убрать свой `background`-градиент; если док не fixed/з-
  индексный (пикер) — задать ему `z-index`, чтобы `z:-1` лёг внутри него, а не за списком.
- **Клавиатура iOS** — высоту ловить через `visualViewport` (прятать док при открытии, показывать
  с задержкой ~350мс при закрытии, чтобы возврат не попал в анимацию клавиатуры).
- **Тема** — `color-scheme: dark`, тёмная. Проект тёмный by design.
