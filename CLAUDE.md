# TRPG — гайд для Claude Code

Геймифицированный фитнес-трекер, Telegram Mini App. Стек: Vite + React 18 +
React Router 6 + Supabase + Telegram WebApp SDK. Деплой — Vercel. Стили — CSS (НЕ Tailwind).
Пользователь (Дмитрий) — нон-программист, общение на русском.

## Скилы проекта (`.claude/skills/`)

Подхватываются автоматически. **trpg-workflow** — входная точка для любого запроса по проекту;
из неё маршрутизация в остальные:

- **trpg-workflow** — рабочий процесс: как вести задачу, формат правок, коммиты+пуш,
  именование, грабли, добавление программ, обновление самих скилов.
- **trpg-supabase** — конвенции БД: RPC, RLS, миграции, Edge Functions.
- **trpg-ui** — дизайн-система: CSS-переменные, скругления, цвета, шрифты, press-эффекты,
  компоненты, анимации, хаптика Telegram.

## Чтение файлов — точечно

Под задачу читать только относящиеся файлы + их прямые связи, а не весь репозиторий.
Дерево проекта в trpg-workflow — карта для навигации, не повод читать всё.

## Коммиты + пуш

Дмитрий разрешил **авто-коммит и авто-push** в этом репо: после `npm run lint` + `npm run build`
(оба зелёные) коммит прямо в `main` (сообщение на русском, conventional commits) и сразу
`git push`. Ручное нажатие в GitHub Desktop не нужно. Проверка — на сайте GitHub; если что —
откат по просьбе.

## Context7 (актуальная документация библиотек) — экономно

Context7 — источник свежей официальной документации внешних библиотек/SDK/API. Использовать
**только когда правда нужно**, а не на каждом шаге (беречь токены).

**Когда вызывать:** новый код или интеграция с React, React Router, Vite, Supabase (client SDK,
RLS, auth, realtime), Telegram Mini Apps / Bot API, любым сторонним npm-пакетом — и когда важно
именно текущее поведение API (версии, устаревшие методы, breaking changes). Тянуть только нужный
кусок доки под задачу, не всю библиотеку.

**Когда НЕ вызывать:** UX/UI и продуктовые решения, ревью дизайна, архитектура, бизнес-логика,
чистый JS/TS без внешних API, рефакторинг без смены API библиотек.

**Ключевое правило (экономия):** если я уверен в API и свежие изменения вряд ли на него влияют —
Context7 НЕ дёргать, работать сразу. Вызывать только для проверки неуверенного или
version-sensitive поведения. Приоритет при расхождениях: доки Context7 → конвенции проекта →
знания модели; изменение по свежей доке — коротко пояснять.

**Ревью кода:** определить используемые библиотеки → по ним свериться с Context7 → сообщить
находки (устаревшие/неверные вызовы, breaking changes, лучшие паттерны) → править только после
согласования.

_Прим.: коннектор Context7 подключается как MCP-сервер в Claude Code (`claude mcp` / настройки).
Если он не подключён в сессии — эти правила ждут своего часа, вызвать его нельзя._
